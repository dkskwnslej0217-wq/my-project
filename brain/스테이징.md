# 스테이징 시스템

## 구조
콘텐츠 생성 → output/staging/ 대기 → "업로드 시작해줘" → 재구성 → Make.com → output/uploaded/

## 폴더 역할
- output/staging/ → 생성 완료, 업로드 대기 중
- output/uploaded/ → 업로드 완료 보관

## 스테이징 저장 형식
파일명: [날짜]_[플랫폼]_[주제].md

파일 내부 헤더 (반드시 포함):
```
---
platform: threads | instagram | youtube
topic: [주제]
score: [점수]
status: staging
created: [날짜]
---
[콘텐츠 본문]
```

---

## 트리거: "업로드 시작해줘"

### 1단계 — 목록 파악
output/staging/ 파일 전체 읽기
플랫폼별로 분류:
- Threads 목록
- Instagram 목록
- YouTube 목록

### 2단계 — 업로드 가능 여부 판단
각 파일 헤더 확인:
- score ≥ 7 → 업로드 대상
- score < 7 → 목록에 표시하되 junho에게 확인 요청
- 영상 필요 플랫폼 (Instagram/YouTube) → brain/영상생성.md 체크리스트 확인
  - 체크리스트 ✅ → 영상 생성 후 업로드
  - 체크리스트 ❌ → "영상 미준비 — 스크립트만 저장, 수동 업로드 필요" 표시

### 3단계 — 플랫폼별 재구성
각 파일을 해당 플랫폼 스펙에 맞게 재포맷:
- Threads → skills/스레드_v4.md 포맷 적용
- Instagram → skills/인스타그램_v4.md 포맷 적용
- YouTube → skills/유투브_v4.md 포맷 적용

### 4단계 — 업로드 제한 확인 (자동업로드.md 규칙)
- 유튜브: 하루 최대 5개 초과 시 다음 날로 예약
- 인스타: 하루 최대 3개 초과 시 다음 날로 예약
- 새벽 2시~6시 → 업로드 금지, 큐에 대기

### 5단계 — Make.com 웹훅 호출
```bash
source .env
# Threads
curl -X POST $MAKE_THREADS_WEBHOOK \
  -H "Content-Type: application/json" \
  -d "{\"title\":\"[제목]\",\"content\":\"[본문]\",\"platform\":\"threads\",\"date\":\"[날짜]\"}"

# Instagram (영상 있을 때만)
curl -X POST $MAKE_INSTAGRAM_WEBHOOK \
  -H "Content-Type: application/json" \
  -d "{\"title\":\"[제목]\",\"script\":\"[스크립트]\",\"platform\":\"instagram\",\"date\":\"[날짜]\"}"

# YouTube (영상 있을 때만)
curl -X POST $MAKE_YOUTUBE_WEBHOOK \
  -H "Content-Type: application/json" \
  -d "{\"title\":\"[제목]\",\"script\":\"[스크립트]\",\"platform\":\"youtube\",\"date\":\"[날짜]\"}"
```

### 6단계 — 완료 처리
- 웹훅 성공 → 파일 헤더 status: uploaded 로 변경 → output/uploaded/ 이동
- 웹훅 실패 → status: failed 유지 → brain/에러알림.md → 텔레그램 알림
- 완료 보고 (3줄 이내):
  ```
  업로드 완료: Threads 3개 / Instagram 1개 / YouTube 0개
  실패: 0개
  대기 중: 2개 (영상 미준비)
  ```
