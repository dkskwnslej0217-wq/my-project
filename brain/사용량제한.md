# 사용량 제한 + 수익화 구조

## 목표
무료/유료 플랜 구분해서
사용량 제한으로 자연스럽게 결제 유도

## 플랜 구조

### 무료 플랜 (Free)
- 하루 3회 실행
- 기본 플랫폼 1개만 (스레드만)
- 자기점수 없음
- 예측엔진 없음
- 광고 문구 포함

### 스타터 플랜 (월 1만원)
- 하루 10회 실행
- 플랫폼 2개 (스레드 + 인스타)
- 자기점수 포함
- 예측엔진 포함

### 프로 플랜 (월 3만원)
- 무제한 실행
- 모든 플랫폼
- 모든 기능 사용
- 우선 지원

### 팀 플랜 (월 10만원)
- 멀티유저 5명
- 모든 기능
- 구글시트 연동
- 전용 지원

## 사용량 추적
Supabase users 테이블:
- user_id
- plan_type (free/starter/pro/team)
- daily_count (오늘 실행 횟수)
- monthly_count (이번 달 실행 횟수)
- last_reset (마지막 초기화 날짜)
- paid_until (결제 만료일)

## 실행 시 체크
run_core.md 맨 앞에:
1. users 테이블에서 user_id 조회
2. 오늘 실행 횟수 확인
3. 플랜별 한도 초과 시:
   → "오늘 무료 한도 3회 모두 사용했어요"
   → "스타터 플랜 업그레이드하면 10회 가능해요"
   → 결제 링크 안내

## 결제 연동 (나중에)
- 카카오페이 or 토스페이먼츠
- 결제 완료 시 users 테이블 plan_type 자동 업데이트
- Make.com으로 결제 알림 자동화
