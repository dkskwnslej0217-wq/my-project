## 캐싱 시스템 (토큰 절감 핵심)

### 1. 완전 동일 요청 캐싱
```
요청 들어옴
↓
날짜 + 주제 + 플랫폼 해시 생성
↓
Supabase cache 테이블 조회
↓
히트 → 즉시 반환 (API 0회, 토큰 0)
미스 → 생성 후 저장 (24시간 유효)
```

### 2. 유사 요청 캐싱
- 주제 유사도 90% 이상 → 캐시 결과 + 훅만 변형해서 반환
- 절감: 95%

### 3. 배치 처리
- 같은 시간대 유사 요청 10개 → API 1회 호출 → 결과 분리
- 절감: 90%

### 4. Claude 프롬프트 캐싱
- Claude API 자체 기능 (시스템 프롬프트 5분 캐시)
- 반복 호출 시 토큰 90% 할인

### 캐시 테이블 (Supabase 추가 필요)

## 전체 요청 처리 흐름

```
유저 요청
↓
1. run_safe.sh 보안 체크
2. 플랜 확인 + 한도 체크 (유저 수 기반 자동 조절)
3. 캐시 확인 → 히트 시 즉시 반환 (끝)
4. 복잡도 판단 → 모델 자동 선택
5. 배치 가능 여부 확인
6. 성공 예시 자동 첨부 (Supabase score 8↑)
7. 생성
8. 유저별 Supabase 저장
9. 성공 예시 풀 자동 업데이트
10. 워터마크 추가 (무료 유저)
11. staging/ 저장 → 업로드 대기
```

유저 1,000명이 써도 API 비용이 100명 수준으로 유지됨
